# 90 Days DevOps Learning Plan

## Current Level

I am an aspiring DevOps Engineer currently learning through hands-on practice. I have basic knowledge of Linux, Git, GitHub, AWS EC2, Docker, and shell scripting. My goal is to become job-ready by building real-world DevOps projects and improving my problem-solving skills.

---

## My Understanding of DevOps & Cloud

DevOps is a culture and engineering practice that combines development and operations to deliver software faster, more reliably, and with automation.

Cloud platforms like AWS provide scalable infrastructure, allowing applications to run without managing physical servers.

---

## Why I Started Learning DevOps

I want to build a career in DevOps because I enjoy solving technical problems and working with cloud technologies.

I also want to:
- Build production-ready projects.
- Get a remote DevOps job.
- Continuously improve my technical and communication skills.

---

## My Goals for the Next 90 Days

1. Build strong Linux, Git, Docker, and AWS fundamentals.
2. Learn Kubernetes, Terraform, and CI/CD by completing hands-on projects.
3. Build a complete production-ready DevOps project and publish my journey on GitHub and LinkedIn.

---

## Core Skills I Want to Build

- Linux troubleshooting
- CI/CD Pipeline Automation
- Kubernetes & Cloud Deployment

---

## Weekly Time Commitment

Monday – Friday:
- 2–3 hours every day

Saturday & Sunday:
- 5–6 hours for projects, revision, and documentation

---

## How I Will Stay Consistent

- Practice every day, even if only for one hour.
- Push code to GitHub daily.
- Share weekly progress on LinkedIn.
- Focus on understanding concepts instead of memorizing commands.
- Never skip learning two days in a row.

---

## My Target After 90 Days

Become confident in Linux, Docker, Kubernetes, AWS, Terraform, and CI/CD, complete multiple real-world projects, and be ready to apply for DevOps Engineer internships and entry-level roles.
